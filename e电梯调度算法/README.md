# 电梯调度算法

基于 LOOK 算法的单部 10 层电梯调度模拟系统。

## 项目简介

本项目实现了经典 LOOK 电梯调度算法，模拟单部 10 层电梯的运行调度。支持批量接收乘客上下楼请求，自动生成最优运行路径。

## 功能特性

- 单部 10 层电梯运行模拟
- 支持外部请求（电梯外按钮）和内部请求（电梯内目标楼层）
- LOOK 智能调度算法
- 自动处理请求去重
- 初始方向智能判定
- 完整运行路径可视化输出

## 算法说明

### 为什么选择 LOOK 算法？

| 算法 | 优点 | 缺点 |
|------|------|------|
| FCFS | 实现简单 | 频繁往返，运行距离长 |
| SSTF | 单次响应快 | 存在"饥饿"问题 |
| SCAN | 公平性好 | 必须到达端点，产生空驶 |
| **LOOK** | 消除空驶，效率最优 | - |

**LOOK 算法核心原则：**
- 电梯沿一个方向移动，处理该方向所有请求
- 到达最远请求后立即折返，不需到达端点
- 折返后处理另一方向的请求

## 使用方法

### 编译

```bash
gcc -o elevator elevator.c
```

### 运行

```bash
./elevator
```

### 输入流程

1. 输入外部请求总数量
2. 输入电梯初始楼层
3. 批量输入外部楼层请求（空格分隔）
4. 按顺序输入每个外部请求对应的目标楼层

## 核心数据结构

```c
typedef struct {
    int currentFloor;          // 当前所在楼层（1-10层）
    int externalRequests[10]; // 外部请求队列
    int internalRequests[10]; // 内部目标请求队列
    int direction;            // 运行方向：0向上，1向下
    int finalTargets[20];     // 最终目标楼层队列
} Elevator;
```
